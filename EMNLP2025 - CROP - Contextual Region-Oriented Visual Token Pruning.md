# CROP: Contextual Region-Oriented Visual Token Pruning 论文总结

## 1. 研究动机
当前基于视觉语言模型(VLM)的视觉问答(VQA)方法通常处理整个图像，导致产生大量包含冗余信息的视觉token。这些不必要的图像细节创建了许多视觉token，极大地增加了VLMs的内存和计算需求。例如，LLaVA-1.5中有576个视觉token，而LLaVA-NeXT对于672×672的图像则有2880个视觉token。这种冗余信息不仅影响效率，还可能干扰模型对关键信息的关注。

## 2. 解决问题
论文致力于解决的核心问题是：如何减少视觉语言模型中不必要的视觉token，同时保持或提高模型在视觉问答任务中的性能。具体来说，作者希望：
1. 减少视觉token的数量，降低内存和计算需求
2. 保留与问题相关的关键视觉信息
3. 不损害模型在VQA任务中的表现

## 3. 现象分析
作者观察到以下现象：
- 现有的视觉token修剪方法主要依赖图像的内在语义或LLM中的注意力图，但没有考虑输入的问题，可能会忽略任务相关的关键信息
- 对于特定问题，我们只需要定位支持答案的局部图像区域，而不是处理整个图像
- 现有的修剪方法导致离散和碎片化的视觉token，破坏了视觉区域的空间语义和连续性
- VLMs在早期层更依赖于明确的视觉token，而在更深层次中信息变得越来越抽象和多模态融合

## 4. 主要方法
论文提出了CROP (Contextual Region-Oriented Visual Token Pruning) 框架，这是一个两阶段的流程：

### 4.1 定位阶段 (Localization)
- 使用一个高效的模型(Qwen2VL-2B)识别与输入查询相关的上下文区域
- 在8×8网格上表示图像，上下文区域定义为块坐标[xmin, ymin, xmax, ymax]
- 定位模块作为即插即用组件，约需8小时训练

### 4.2 修剪阶段 (Pruning)
#### a) Pre-LLM Compression (PLC)
- 在输入LLM之前自适应压缩不同图像区域
- 对上下文区域使用较低的压缩比，对非上下文区域使用较高的压缩比
- 使用可学习的查询压缩上下文区域和非上下文区域的token
- 定义锚点token保留上下文区域的精细细节

#### b) Inner-LLM Pruning (ILP)
- 一种无需训练的方法，在早期LLM层中修剪token
- 在指定的早期层K直接移除上下文区域外的视觉token
- 无需重新训练大型VLM，即插即用

## 5. 数据与实验
### 5.1 数据集
- 训练数据集包括区域定位样本(410k)和LLaVA指令调优数据(200k)，总共610k样本
- 使用的数据集包括TextVQA、Flickr30k、DocVQA、VSR、GQA、OpenImage、CUB、V7W等
- 测试在多个基准上进行，包括GQA、MME、VQAT、SQA、VQA V2、POPE等

### 5.2 基线模型对比
- FastV
- SparseVLM
- PDrop
- VisionZip
- LLaVA-1.5和LLaVA-NeXT作为完整模型的上限

### 5.3 核心实验结果
- 在LLaVA-1.5上，ILP策略在66.7%视觉token修剪率下，LLaVA-1.5-7B和LLaVA-1.5-13B分别保留了98.9%和99.8%的基线性能
- 在88.9%的修剪率下，这些模型保持了96.0%和96.2%的原始效率，且无需微调
- PLC策略在88.9%视觉token修剪率下，分别保留了95.4%和95.8%的性能
- 在LLaVA-NeXT上，CROP在94.4%的修剪率下仍保持了94.3%的性能
- CROP在多个基准测试上取得了最先进的性能，同时显著减少了视觉token的数量

## 6. 主要贡献
论文的主要贡献包括：
1. 开发了一个高效的定位模型来识别上下文区域，作为即插即用组件指导视觉token修剪
2. 引入了一种新颖的基于上下文区域的视觉token修剪框架，并开发了两种不同的修剪方法：PLC和ILP
3. 据作者所知，这是第一个将上下文区域引入视觉token修剪的工作
4. 实验结果表明，所提出的方法在无需任何训练或微调的ILP方法中取得了最先进的性能
5. 通过保留连续的上下文区域，而不是离散的、碎片化的token，维护了视觉区域的空间语义和连续性
