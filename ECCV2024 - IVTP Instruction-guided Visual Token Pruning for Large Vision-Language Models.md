# IVTP: Instruction-guided Visual Token Pruning for Large Vision-Language Models 论文总结

## 1. 研究动机
- **计算成本问题**：大型视觉语言模型(LVLMs)虽然取得了显著进展，但冗长的视觉标记(visual tokens)导致高昂的计算成本和标记预算占用
- **现有方法局限性**：
  - 一些方法与特定模型架构紧密耦合，难以迁移到其他框架
  - 传统视觉任务标记剪枝方法直接应用于LVLMs时表现不佳，因为LVLMs通常冻结视觉编码器，无法进行端到端优化
  - 单层的重要性评估不够稳定，难以准确评估标记重要性

## 2. 解决问题
- **核心问题**：如何在保持LVLMs性能的同时减少视觉标记数量，降低计算成本
- **具体挑战**：
  - 设计一种与冻结视觉编码器兼容的标记剪枝方法
  - 提高方法的迁移性和稳定性
  - 根据不同文本指令自适应调整视觉信息
  - 有效过滤与指令无关的视觉标记，避免引入噪声信息

## 3. 现象分析
- **关键观察**：并非所有视觉标记都对最终响应同等重要，冗余标记导致不必要的计算开销
- **问题分析**：
  - 传统视觉标记剪枝方法在LVLMs中表现不佳，因为视觉编码器通常被冻结
  - 单层的重要性评估缺乏稳定性，难以准确评估标记重要性
  - 处理不同视觉指令时，相关视觉标记的重要性会动态变化
  - 直接聚合所有文本标记来指导视觉标记选择会引入噪声信息

## 4. 主要方法
- **IVTP(Instruction-guided Visual Token Pruning)**：一种两阶段视觉标记剪枝方法

### 4.1 组级标记剪枝(GTP)模块
- **核心机制**：基于注意力展开(attention rollout)的组级标记重要性评估
- **工作原理**：
  - 将网络分成多个组，在组内通过注意力展开聚合相邻层注意力信息
  - 在每组顶层执行标记剪枝
  - 使用CLS行注意力权重加权补丁标记归一化值，计算重要性分数
- **优势**：注意力展开得到的分数与对预测输出的贡献相关性更高，使标记决策更准确

### 4.2 LLM中的指令引导标记剪枝
- **核心机制**：基于文本伪CLS标记的视觉标记重要性评估
- **工作原理**：
  - 利用CLIP文本编码器处理文本指令，获取文本标记序列
  - 计算文本标记与视觉CLS标记的余弦相似度，确定文本视觉相关性
  - 提取关键文本标记形成文本伪CLS标记
  - 在LLM中重用GTP模块，基于文本伪CLS标记的注意力权重进行剪枝

### 4.3 两阶段标记剪枝
- **第一阶段**：在ViT中，基于视觉CLS标记和GTP剪枝低信息含量标记
- **第二阶段**：在LLM前W层中，基于文本伪CLS标记和GTP剪枝与当前指令无关的视觉标记

## 5. 数据与实验

### 5.1 数据集
- **预训练数据**：LAION-CC-SBU数据集的558K子集，配合BLIP标题
- **指令微调数据**：包含665K指令示例的多样化数据集
- **评估基准**：12个标准测试集，包括VQAv2、GQA、VisWiz、SQAI、VQAT、POPE、MME、MMB、MMBCN、SEED、LLaVAW、MM-Vet

### 5.2 对比基线模型
- **简单标记方法**：随机采样、基于相似度的TopK、基于空间相关的子采样
- **视觉任务SOTA方法**：EViT、ToMe
- **LVLMs专用方法**：Honeybee、LLaMA-ViD、Qwen-VL

### 5.3 核心实验结果
- **视觉标记减少88.9%**（从576减少到64）
- **计算复杂度降低超过46%**
- **平均准确率仅下降1.0%**
- **显著优于所有对比方法**，在准确率和计算效率间取得最佳平衡

## 6. 主要贡献
1. **提出创新的两阶段视觉标记剪枝方法IVTP**：
   - 显著减少视觉标记数量，同时保留关键视觉信息
   - 在标记减少88.9%的情况下，仅带来1.0%的平均准确率下降
   - 计算复杂度降低超过46%

2. **提出可在ViT和LLM中兼容工作的GTP模块**：
   - 基于注意力展开实现组内注意力聚合，改进视觉标记重要性评估
   - 引入语义相关的伪CLS标记，过滤与当前文本指令相关的视觉标记
   - 使重要性评估对冻结的视觉编码器更加有效

3. **广泛的实验验证**：
   - 在12个标准基准测试上验证方法有效性
   - 证明方法在准确率和计算效率间取得优越平衡
   - 显著优于最先进的标记剪枝方法

4. **良好的迁移性和稳定性**：
   - 不需要重新训练，可直接集成到现有LVLMs中
   - 与特定模型架构解耦，易于迁移到其他框架
