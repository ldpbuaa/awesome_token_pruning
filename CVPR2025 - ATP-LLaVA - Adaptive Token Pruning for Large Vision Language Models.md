# ATP-LLaVA: Adaptive Token Pruning for Large Vision Language Models 论文总结

## 1. 研究动机
- 大型视觉语言模型(LVLMs)在多模态任务中取得显著成功，但在资源有限设备上处理大量视觉标记(visual tokens)的计算成本过高
- 现有方法已识别到LLM解码器层中视觉标记存在冗余，通过预定义或固定比率修剪标记减少计算开销
- 作者观察到修剪比率在不同LLM层和不同实例(图像-提示对)间的影响存在差异
- 需要开发逐层(layer-wise)和实例级(instance-wise)的视觉标记修剪策略，以平衡计算成本与模型性能

## 2. 解决问题
- 如何实现自适应的视觉标记修剪策略而非使用预定义固定比率
- 如何在减少计算开销的同时保持模型性能
- 如何在大型视觉语言模型中有效平衡计算效率和视觉理解能力

## 3. 现象分析
- **层间差异现象**：实验发现修剪比率的影响是逐层(layer-wise)的，浅层对修剪更敏感，深层具有更强鲁棒性
- **任务差异现象**：修剪比率的影响也是实例级(instance-wise)的，细粒度任务(如实例计数、空间关系)需保留更多标记，而粗粒度任务(如场景理解)在高修剪比率下性能损失较小
- **最优比率差异**：每个实例和每层的最优修剪比率不同，预定义比率会导致次优模型性能和效率
- **空间建模破坏**：视觉标记修剪会破坏空间建模，这对视觉理解至关重要

## 4. 主要方法
- **ATP-LLaVA框架**：自适应标记修剪方法，动态确定每个LLM层的实例特定标记修剪比率
- **自适应标记修剪(ATP)模块**：
  - 可无缝集成在任何两个LLM层之间，计算开销可忽略
  - 基于输入实例自适应计算重要性分数和修剪阈值
- **空间增强修剪(SAP)策略**：
  - 从标记冗余和空间建模两个角度修剪视觉标记
  - 包括冗余修剪(基于自模态和跨模态重要性)和空间修剪(空间维度均匀采样)
- **可学习阈值**：使用具有双预测头的MLP学习特定于实例的阈值，实现逐层和实例级自适应修剪
- **ATP-Loss函数**：平衡标记修剪效率和理解能力，在有限修剪约束下训练模型

## 5. 数据与实验
- **使用的数据集**：
  - GQA, MMB (MMBench), MME, POPE, SEED-Bench, SQA_I, VQA_v2
  - 训练数据：665k视觉指令跟随数据
  - 基础模型：LLaVA1.5，CLIP-ViT-L视觉编码器，Vicuna-7B-1.5语言模型

- **对比的基线模型**：
  - PruMerge+：预定义修剪比率方法
  - FastV：预定义修剪比率方法
  - SparseVLM：预定义修剪比率方法
  - 上限模型：LLaVA-1.5(无修剪)

- **核心实验结果**：
  - ATP-LLaVA将平均标记数减少75%(从576减少到144)，保持98.1%性能，七个基准测试仅下降1.9%
  - 进一步减少到88个标记(平均压缩率约84.7%)，仍保持94.6%性能
  - 在MMBench和SEED-Bench上达到甚至超过上限模型性能
  - 与预定义修剪比率方法相比，相同平均标记数下表现更好，特别是在MMB、GQA、VQA_v2和SEED等测试上

## 6. 主要贡献
- **新发现**：揭示了自适应确定实例级和LLM层级修剪比率对有效视觉标记修剪的重要性
- **新方法**：提出ATP-LLaVA框架，动态减少大型视觉语言模型计算成本
- **新技术**：
  - 引入自适应标记修剪(ATP)模块和ATP-Loss函数，平衡修剪效率和模型能力
  - 提出空间增强修剪(SAP)方法，在修剪过程中保持空间建模
- **显著成果**：实现75%平均修剪比率，同时在七个广泛使用的视觉理解基准测试上保持98.1%性能，仅下降1.9%
