# 论文总结：Walk and Read Less: Improving the Efficiency of Vision-and-Language Navigation via Tuning-Free Multimodal Token Pruning

## 1. 研究动机
作者进行这项研究是因为大型模型在Vision-and-Language Navigation (VLN)任务上表现良好，但在资源受限环境中运行成本高昂。Token pruning通过减少模型输入大小来提高效率，提供性能与成本的权衡。然而，先前工作忽视了VLN特有的挑战：信息损失可能导致代理回溯，增加导航长度和计算成本，从而抵消剪枝带来的效率提升。因此，无法识别无信息标记会破坏剪枝的效率增益。

## 2. 解决问题
论文致力于解决的核心科学问题是如何在保持VLN任务导航成功率(Success Rate, SR)的同时，显著提高计算效率。具体而言，作者提出Navigation-Aware Pruning (NAP)框架，通过三个方面解决效率问题：
- 减少视觉输入("walk less")
- 优先剪枝导航无关的指令标记("read less")
- 改善效率-成功率权衡曲线

## 3. 现象分析
作者观察到几个关键现象：
1. 现有标记剪枝策略是为通用视觉语言模型设计的，忽视了VLN任务中固有的时间依赖性
2. 被剪枝的标记可能包含有助于代理决策的有用上下文，移除它们会导致代理回溯未访问节点，增加路径长度
3. 基于注意力的文本剪枝无法有效区分相关和不相关的指令标记，VLN模型倾向于为标点和功能词分配高注意力分数
4. 背景视图提供的信息高度适合剪枝，而行动视图不适合剪枝

## 4. 主要方法
论文提出了Navigation-Aware Pruning (NAP)框架，包含三个核心组件：

### 4.1 背景剪枝(BackGround Pruning, BGP)
- 将全景视图分为行动视图(action views)和背景视图(background views)
- 只剪枝背景视图，保留所有行动视图
- 通过自注意力机制计算每个视图的重要性分数，剪除重要性最低的背景视图标记
- 极大减少视觉输入同时最小化成功率损失

### 4.2 回溯剪枝(BackTracking Pruning, BTP)
- 跟踪未访问节点的重要性分数
- 在每一步开始时，保留基于最新重要性分数的前k_BTP个未访问节点，移除其余节点
- 减少历史输入大小，限制回溯选项，提高路径效率

### 4.3 词汇优先剪枝(Vocabulary Priority Pruning, VPP)
- 使用大型语言模型(如LLaMA 3)构建"不相关性词汇表"
- 提示LLM识别与导航方向/标题、环境描述或室内/外物体无关的词
- 在语言transformer编码器中，首先过滤掉不相关性词汇表中的词，若不足则基于注意力分数进一步剪枝
- 有效保留指令中的关键词，即使在高剪枝率下也能使代理正常工作

## 5. 数据与实验
### 5.1 数据集
- R2R (Anderson et al., 2018)
- RxR-English (Ku et al., 2020)
- REVERIE (Qi et al., 2020)

### 5.2 对比的基线模型
- 随机剪枝(Random pruning)
- 级联剪枝(Cascade pruning)
- FastV (Chen et al., 2025)
- Token Merging (ToMe) (Bolya et al., 2022)

### 5.3 测试的VLN模型
- HAMT (Chen et al., 2021)
- DUET (Chen et al., 2022)
- GOAT (Wang et al., 2024b)

### 5.4 核心实验结果
- NAP在多个VLN模型和数据集上实现显著FLOPS减少，同时保持与基线相当或更高的SR
- 在R2R数据集上，50%标记预算下，NAP比基线方法降低FLOPS多达7个百分点，同时提高SR高达2.5%
- 在所有R2R和REVERIE设置中，NAP以最少的步骤完成导航
- NAP实现了比原始模型高2倍的加速，比最先进的剪枝基线高1.25倍的加速，同时成功率下降更小(23个百分点)
- 效率-成功率权衡曲线显示，NAP在相同成功率损失下比FastV实现更高的效率增益(例如，在RxR数据上，NAP实现14个百分点的效率增益)

## 6. 主要贡献
这篇论文对该领域的主要贡献包括：

1. **提出首个VLN特定剪枝框架NAP**：包含背景剪枝(BGP)减少视觉输入，回溯剪枝(BTP)提高路径效率，解决了现有方法忽视的VLN特有时间依赖性问题。

2. **引入词汇优先剪枝(VPP)**：利用大型语言模型构建"不相关性词汇表"，识别并剪枝导航无关的指令标记，同时保留关键信息，解决了基于注意力剪枝的噪声问题。

3. **显著的效率-性能权衡改进**：通过结合BGP、BTP和VPP，NAP实现了比原始模型高2倍的加速，比最先进的剪枝基线高1.25倍的加速，同时成功率下降更小。

4. **广泛的验证与通用性**：这些改进在多个VLN模型和数据集上得到验证，包括离散和连续环境，展示了方法的通用性和有效性。

5. **调优自由方法**：NAP允许同一模型通过调整剪枝率灵活适应不同硬件约束，无需昂贵的重新训练，适合资源受限环境。
