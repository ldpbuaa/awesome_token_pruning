# 论文深度总结：AIM: Adaptive Inference of Multi-Modal LLMs via Token Merging and Pruning

## 1. 研究动机
- **背景**：多模态大语言模型(MLLMs)在视觉理解方面表现出色，但它们通常依赖视觉编码器生成的大量视觉token，导致计算需求高。
- **痛点**：高计算需求限制了这些模型在资源受限环境中的应用，以及处理长上下文任务的能力。特别是对于视频数据，每个视频的token数量可能达到数千，随着视频帧数增加，token总数也会增长，导致模型处理密集视频帧的能力受限，常常丢失关键的时序信息。

## 2. 解决问题
论文致力于解决多模态大语言模型的高计算需求问题，具体是：
- 如何减少视觉token的冗余，降低计算负载
- 如何实现自适应推理，使模型能够根据不同的计算资源需求动态调整计算量
- 如何在减少计算量的同时保持模型性能，甚至提升长视频理解能力

## 3. 现象分析
作者观察到的关键现象包括：
- 视觉数据中存在固有的冗余性，不需要保留所有视觉token即可维持接近的性能
- 仅需25%的视觉token输入LLM就能维持接近的性能
- 减少每帧的token数量可以使LLM处理更多帧，从而解决信息丢失问题，提高长视频理解性能
- 在LLM早期层修剪文本token或移除视觉token会显著影响性能，而在后期层修剪大量视觉token可以保持性能
- 多模态LLM在早期层更注重跨模态融合，在后期层更优先处理文本token

## 4. 主要方法
论文提出了一种训练自由的自适应推理方法，包括两个核心组件：

**a) 基于嵌入相似性的迭代token合并**
- 在LLM之前，根据token嵌入之间的余弦相似性合并高度相似的视觉token
- 将相邻token分为集合A和B，计算集合间的相似度分数，识别每个A集合token在B集合中最匹配的token
- 通过平均嵌入合并相似度最高的token对，最多可将token数量减半
- 该过程可迭代重复（如两次）以达到期望的保留比例

**b) 基于多模态重要性的渐进式token修剪**
- 在LLM各层中，基于自注意力权重计算每个token的重要性分数
- 使用PageRank算法将注意力权重作为邻接矩阵来计算token的重要性分数
- 只修剪视觉token，保留最重要的视觉token，保持文本token完整
- 设计了一个调度器来控制每层的保留比例，作为分段函数，其中参数l1和l2控制修剪进度

## 5. 数据与实验
**数据集**：
- 视频基准：VideoMME、MLVU、EgoSchema、MVBench、NextQA、PerceptionTest
- 图像基准：GQA、VQAv2、MME、TextVQA、SQA-IMG、MMB、POPE

**对比的基线模型**：
- FastV：在特定LLM层修剪视觉token
- VTW：在特定选择的LLM层放弃所有视觉token
- PDrop：将LLM层分为四个阶段，在每个阶段结束时基于视觉token和最后一个文本token之间的注意力分数修剪token
- LLaVA-Prumerge：利用视觉编码器中的键值对来修剪和合并视觉token

**核心实验结果**：
- 视频LLM：与基础模型LLaVA-OV-7B相比，FLOPs减少6.8倍，prefill时间减少8.0倍，性能几乎没有下降
- 在相同计算资源下，方法允许处理更多帧，甚至在长视频理解上超越最先进的视频LLM（如MLVU上提升+4.6）
- 图像LLM：与基础模型LLaVA-1.5-7B相比，FLOPs减少3.7倍，prefill时间减少2.7倍，性能损失可控
- 方法实现了高达40倍的FLOPs减少，同时保持可接受性能损失

## 6. 主要贡献
这篇论文对该领域的主要贡献包括：
- 提出了一种训练自由的自适应推理方法，能够减少视觉token冗余和计算需求，同时保持基础模型性能
- 采用极简设计，在LLM前合并视觉token，并在LLM层内渐进修剪视觉token，适用于各种多模态LLM模型
- 通过自适应推理，实现了计算成本的大幅降低，同时保持了基础图像和视频LLM的性能，并且在相同FLOPs下超越了最先进的视频LLM
- 提供了深入的研究，揭示了视觉token的冗余性和LLM各层的行为，为未来高效多模态LLM的设计提供了指导
