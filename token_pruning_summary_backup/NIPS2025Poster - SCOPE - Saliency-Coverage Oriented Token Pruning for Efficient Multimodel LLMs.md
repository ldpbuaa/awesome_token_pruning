# 论文深度总结：SCOPE: Saliency-Coverage Oriented Token Pruning for Efficient Multimodel LLMs

## 1. 研究动机

- **计算开销问题**：多模态大语言模型(MLLMs)通常处理大量视觉token，即使许多token是冗余的，仍导致显著的计算开销
- **现有方法局限性**：现有视觉token剪枝方法主要基于注意力分数选择最显著token，导致所选token的语义不完整性
- **注意力分布问题**：基于显著性的方法通常遭受高度倾斜的注意力分布，只有少数token获得大量注意力，其余token表现出几乎均匀的注意力值，降低了token区分度
- **实际应用限制**：高分辨率图像和视频场景下token数量激增，限制了模型在边缘计算和机器人等实际应用中的部署

## 2. 解决问题

- **语义完整性保持**：解决现有方法在减少token数量时无法保持语义完整性的问题
- **注意力分布均衡**：克服高度倾斜的注意力分布问题，提高模型区分有价值token和冗余token的能力
- **计算效率优化**：在保持模型性能的同时，显著减少视觉token数量，加速MLLM推理
- **综合剪策策略**：设计一种联合考虑显著性和覆盖范围的视觉token剪枝策略，而非仅依赖单一指标

## 3. 现象分析

- **语义损失现象**：基于显著性的方法倾向于只选择一小部分视觉token(如图1a所示)，导致显著的语义损失
- **注意力倾斜问题**：图1b显示注意力权重分布高度倾斜，只有少数token获得大量注意力，其余token的注意力值几乎平坦
- **覆盖率不足**：图2表明基于显著性的方法在θ-覆盖率指标上表现不佳，甚至比随机选择基线更差
- **性能下降与压缩率关系**：随着token数量减少，所有方法性能都会下降，但SCOPE在相同压缩率下保持更高性能(如图1c所示)

## 4. 主要方法

提出**SCOPE**(Saliency-Coverage Oriented Token Pruning for Efficient Multimodal LLMs)方法，核心机制包括：

- **集合覆盖(Set-coverage)定义**：基于token间余弦相似性，量化所选token集合的语义覆盖程度
  ```
  C(S) = Σ_{v∈V} max_{s∈S} sim(v,s)
  ```

- **Token-coverage增益**：量化通过包含未选择token可获得多少额外覆盖
  ```
  Δ(v;S) = Σ_{u∈V} [max(S_{uv},c_u) - c_u]
  ```

- **SCOPE分数**：整合token显著性分数与token-coverage增益
  ```
  SCOPE(v;S) = Δ(v;S) · A_v^α
  ```
  其中A_v是视觉token的注意力分数，α是缩放因子

- **迭代选择策略**：算法1详细描述了SCOPE的完整流程，包括初始化、计算边际增益、选择下一个token、更新选定子集和覆盖分数等步骤

## 5. 数据与实验

### 数据集

- **视觉语言理解基准**：GQA、MMBench、POPE、ScienceQA、TextVQA、SEEDBench和MMVet
- **视频理解基准**：TGIF、MSVD、MSRVTT和ActivityNet

### 基线模型

- FastV (ECCV'24)
- SparseVLM (ICML'25)
- VisionZip (CVPR'25)
- PDrop (CVPR'25)

### 核心实验结果

- **性能对比**：在LLaVA-1.5 7B模型上，保留64个token(减少88.9%)的情况下，SCOPE实现96.0%的原始性能保留
- **不同模型表现**：在LLaVA-Next 7B模型上，保留160个token(减少94.4%)的情况下，SCOPE实现97.1%的原始性能保留
- **显著优势**：表1和表2显示SCOPE在多个基准测试中一致优于先前方法，特别是在高压缩率下表现更为突出
- **覆盖率提升**：SCOPE在θ-覆盖率指标上显著优于基于显著性的方法(图2)

## 6. 主要贡献

- **问题揭示**：首次系统揭示基于显著性的视觉token剪枝方法的局限性，包括语义完整性和注意力分布问题
- **新方法提出**：提出SCOPE方法，首次联合建模显著性和覆盖范围，有效保持语义完整性
- **技术集成**：将SCOPE集成到代表性MLLMs(如LLaVA-1.5和LLaVA-Next)中，无需额外训练
- **实证优势**：在多个视觉语言基准上证明SCOPE的有效性，实现了计算效率和任务性能之间的最优权衡，例如在9倍token减少的情况下仍保持96%的性能
