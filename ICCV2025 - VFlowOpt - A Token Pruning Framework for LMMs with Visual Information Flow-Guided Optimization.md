# VFlowOpt: A Token Pruning Framework for LMMs with Visual Information Flow-Guided Optimization - 论文总结

## 1. 研究动机

大型多模态模型(LMMs)通过利用大量视觉标记获取细粒度视觉信息，在视觉语言任务中表现出色，但标记冗余导致显著计算成本。现有方法在推理过程中减少视觉标记时，通常依赖于从仅视觉标记或视觉-语言标记的注意力分数导出的重要性图，以在一个或多个剪裁阶段剪裁标记。尽管取得进展，但剪裁框架和策略仍然简单且探索不足，往往导致性能大幅下降。作者旨在解决LMMs中视觉标记冗余导致的计算效率瓶颈问题，特别是在资源受限环境或延迟敏感场景下的部署挑战。

## 2. 解决问题

论文致力于解决的核心科学问题是：如何在减少LMMs中视觉标记数量的同时最小化性能损失。具体而言，作者旨在开发一个更有效的标记剪裁框架，能够：
- 更准确地估计标记重要性
- 通过标记回收和渐进式剪裁策略减少信息损失
- 为不同LMMs定制剪裁策略以更好地保留模型性能

## 3. 现象分析

作者观察到以下关键现象和问题：
- 现有基于注意力的标记重要性评估方法受到冗余标记干扰，冗余标记（如背景区域对应的标记）经常分配不成比例的高注意力给其他相似的冗余标记，降低了重要性估计的可靠性
- 视觉标记在LMM的较浅层中扮演更关键的角色，而在较深层中冗余度趋于增加
- 现有剪裁策略过于简单，缺乏对不同LMMs特性的适应能力，常导致显著性能下降
- 粗粒度的剪裁策略往往因不适合不同LMMs的特性而造成性能大幅下降

## 4. 主要方法

论文提出了VFlowOpt，一个包含三个核心组件的标记剪裁框架：

### 4.1 视觉标记重要性估计
- 识别相对重要标记：基于从所有标记接收到的注意力，定义阈值τ = t·max(A_ij)，其中t是敏感度超参数
- 排除冗余标记，使用相对重要标记的注意力作为更稳健的度量
- 将图像块的信息熵纳入重要性分数：H(V_i) = -Σ(p_k·log(p_k))，其中p_k是块中灰度级k像素的比例
- 重要性分数计算：I_i = softmax(α·Σ(A_ki) + (1-α)·H(V_i))，其中α决定熵项贡献

### 4.2 渐进式剪裁和标记回收
- 采用渐进式剪裁策略，将LMM均匀分为三个阶段
- 每个阶段开始时，根据阶段特定保留比率R=[R1,R2,R3]保留重要性分数较高的视觉标记
- 保留率计算：R = (R1·L1 + R1·R2·L2 + R1·R2·R3·L3)/L
- 标记回收机制：使用方形网格（边长为a）将剪裁标记分组，在每个单元格内，使用重要性分数作为权重的加权平均融合剪裁标记，保留空间结构

### 4.3 剪裁策略优化
- 将LMM最后一层的最终标记视为视觉信息流的代表性结果
- 优化目标：最大化cosine_similarity(h_f, g_s(h_f))，其中h_f是剪裁前最终标记表示，g_s是剪裁策略s的作用
- 使用贝叶斯优化系统探索超参数空间：阈值敏感度t、熵权重α、网格大小a和剪裁比率R1,R2,R3
- 仅需少量未标记数据即可为不同LMMs定制最优剪裁策略

## 5. 数据与实验

### 5.1 数据集
- **图像理解任务**：GQA, VizWiz, ScienceQA-IMG, TextVQA, ChartQA, POPE, MME, MMBench, MMStar, DocVQA（共10个基准）
- **视频理解任务**：SeedBench（视频）和VideoMME（按视频长度分为短、中、长子集）

### 5.2 对比的基线模型
- FastV
- FitPrune
- PDrop
- SparseVLM
- VisionZip

### 5.3 核心实验结果
- **性能保持**：在LLaVA-OneVision-7B上，VFlowOpt可剪裁50%视觉标记同时保持99.9%性能
- **极端剪裁**：保留仅10%标记时仍保持85.5%原始性能，优于最佳基线3.3%
- **效率提升**：剪裁90%标记时减少89% KV-Cache内存使用，推理速度提高3.8倍
- **跨模型泛化**：在LLaVA-NeXT-7B和Qwen2-VL-7B等不同架构上均取得SOTA性能
- **关键优势**：VFlowOpt能针对不同LMMs特性定制剪裁策略，而FitPrune和PDrop在某些模型上会破坏正常输出

## 6. 主要贡献

这篇论文对该领域的主要贡献包括：
1. **新的重要性图推导过程**：结合注意力校准和信息熵，更准确估计视觉标记重要性
2. **渐进式剪裁与回收机制**：通过多阶段剪裁和标记回收策略减少信息损失
3. **视觉信息流引导的优化方法**：将LMM最终标记视为视觉-文本交互的最具代表性信号，通过最小化剪裁前后标记表示差异，为不同LMMs定制最优剪裁策略
4. **显著的效率-性能平衡**：剪裁90%视觉标记同时保持90%性能，减少89%内存使用，加速推理3.8倍
5. **广泛的架构适应性**：成功应用于LLaVA-OneVision、LLaVA-NeXT和Qwen2-VL等多种LMM架构
