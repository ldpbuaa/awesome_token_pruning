# 论文总结：GTPT: Group-based Token Pruning Transformer for Efficient Human Pose Estimation

## 1. 研究动机
- **工业应用挑战**：现有2D人体姿态估计方法在公共基准上表现优异，但参数量大、计算开销高，限制了在工业界的实用性。
- **全身姿态估计难题**：高效人体姿态估计仍存在障碍，特别是对于包含大量关键点(如全身姿态估计)的场景。
- **CNN方法局限性**：当前高效姿态估计方法主要依赖CNN，在处理高分辨率特征图以区分密集关键点(如面部)时存在困难，且难以捕获关键点间的长距离相关性。
- **Transformer潜力未充分利用**：Transformer虽在姿态估计中显示潜力，但现有方法引入冗余，影响效率。

## 2. 解决问题
- **减少模型冗余**：解决关键点token和视觉token在全身姿态估计中的冗余问题。
- **计算效率与性能平衡**：在不牺牲性能的前提下，显著减少计算开销。
- **密集与稀疏关键点处理**：有效处理面部(密集关键点)和身体(稀疏关键点)的差异。
- **长距离相关性建模**：利用Transformer优势建模关键点间的全局关系。

## 3. 现象分析
- **关键点冗余性**：随着关键点数量增加，特别是在高密度区域(如面部)，关键点token的冗余性显著增加。
- **浅层网络特性**：在浅层网络中，同一身体部位的关键点往往关注相似区域，单独建模每个关键点变得高度冗余。
- **注意力分布变化**：浅层网络中关键点token主要关注整个人体，深层网络则只关注关键点周围特定区域。
- **修剪偏差问题**：直接高比例修剪会优先保留易定位关键点的视觉token，忽略难定位关键点。

## 4. 主要方法
### GTPT (Group-based Token Pruning Transformer)
- **粗到细的关键点引入策略**：
  - 初始输入仅包含一个人体token (X<sub>h</sub>)
  - 逐步转换为稀疏关键点token (X<sub>s</sub>)和部分token (X<sub>p</sub>)
  - 最终将部分token转换为密集关键点token (X<sub>d</sub>)

- **关键点分组机制**：
  - 将所有关键点分为三组：头部(head)、上身(upper body)和下身(lower body)
  - 根据关键点密度和区域大小差异化处理不同组

- **多头部分组注意力(MHGA)**：
  - 在组内并行计算注意力，降低计算复杂度
  - 利用共享稀疏关键点token作为全局信息连接各组
  - 保持组间交互的同时控制计算开销

- **全局感知修剪策略**：
  - 基于注意力图修剪冗余视觉token
  - 使用softmax pooling计算视觉token重要性分数
  - 动态调整各组保留的视觉token数量

- **课程学习训练策略**：
  - 第一阶段：训练基础人体姿态估计能力
  - 第二阶段：引入部分token和密集关键点token学习全身姿态估计

## 5. 数据与实验
### 数据集
- **COCO数据集**：用于标准人体姿态评估
- **COCO-WholeBody数据集**：用于全身姿态评估(包含身体、面部、手部和足部关键点)

### 基线模型对比
- **CNN-based方法**：HRNet-W32, HRNet-W48, Lite-HRNet-18/30, EfficientPose-B/C
- **Transformer-based方法**：TransPose-R-A4/H-S, TokenPose-S-v1/B, DistilPose-S/L, PPT-S/B

### 核心实验结果
- **COCO验证集表现**：
  - GTPT-S (1.6 GFLOPs, 5.4M参数)：AP 73.6%，优于大多数对比方法
  - GTPT-B (3.6 GFLOPs, 8.3M参数)：AP 74.9%，与TokenPose-B相当但计算量减少40%

- **COCO-WholeBody验证集表现**：
  - GTPT-B (4.0 GFLOPs)：全身AP 59.6%，显著优于其他方法
  - 面部关键点AP 84.2%，手部关键点AP 47.9%，表现尤为突出
  - 在保持较低计算量的同时实现了更高精度

## 6. 主要贡献
- **新架构GTPT**：提出基于分组token修剪的Transformer框架，通过关键点分组和视觉token修剪显著减少冗余，提高全身姿态估计效率。
- **MHGA注意力机制**：设计多头部分组注意力，在保持低计算开销的同时实现组内精细建模和组间全局交互。
- **粗到细引入策略**：创新性地采用逐步引入关键点的方法，有效缓解大量关键点对计算效率的影响。
- **实用高效解决方案**：实验证明GTPT在保持高性能的同时显著降低计算复杂度，特别适合工业部署和实时应用场景。
